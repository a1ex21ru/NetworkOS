## 1. Задание

Некий университет решил продемонстрировать свою политкорректность, применив известную доктрину верховного суда США "Равенство порознь равенством не является" не только к цвету кожи, но и к полу. Результатом этого решения явились совместные ванные комнаты в общежитиях. Тем не менее в поддержку исторически сложившейся традиции университет постановляет, что если в ванной комнате есть женщина, то другая женщина может туда зайти, а мужчина не может, и наоборот. На двери ванной есть индикатор, показывающий, в каком из трех состояний находится ванная:

• никого нет;
• в ванной женщины;
• в ванной мужчины.

Однако число кабинок в ванной ограничено. Поэтому, если все места в ванной заняты, то туда не может войти ни мужчина, ни женщина. Таким образом, должно также отображаться число свободных мест в ванной комнате. 

Ваша задача – написать программу, в которой каждый клиент (будь то мужчина или женщина) представляет собой отдельный процесс или отдельный поток (в зависимости от способа реализации), а ванная комната – разделяемый ресурс. Организовывать очередь для процессов или потоков в программе не требуется: то, в какой последовательности потоки будут пытаться завладеть ресурсом, будет зависеть от планировщика операционной системы.

## 2. Структуры данных

### 2.1. Перечисление состояний

```c
typedef enum { 
    EMPTY,   // Ванная комната свободна
    MALE,    // В ванной находятся только мужчины
    FEMALE   // В ванной находятся только женщины
} BathroomStatus;
```

**Назначение:** Определяет текущее состояние ванной комнаты.

### 2.2. Структура Bathroom

```c
typedef struct {
    pthread_mutex_t access_lock;          // Мьютекс для защиты критической секции
    pthread_cond_t  availability_signal;  // Условная переменная для ожидания
    
    unsigned int total_cabins;            // Общее количество кабинок
    unsigned int occupied_cabins;         // Количество занятых кабинок
    
    BathroomStatus current_status;        // Текущий статус ванной (пол внутри)
    BathroomStatus last_gender;           // Пол последнего использовавшего ванную
    
    // Механизм предотвращения голодания
    unsigned int consecutive_entries;     // Последовательные входы одного пола
    unsigned int max_consecutive_entries; // Лимит последовательных входов
    bool forced_switch_pending;           // Флаг: требуется принудительная смена пола
} Bathroom;
```

**Назначение:** Хранит состояние ванной комнаты и средства синхронизации.

**Ключевые поля:**
- `current_status` — текущее состояние индикатора на двери (EMPTY/MALE/FEMALE)
- `forced_switch_pending` — флаг принудительной смены пола после достижения лимита `max_consecutive_entries`
- `last_gender` — позволяет реализовать справедливое чередование при освобождении ванной
- `occupied_cabins` — используется для отображения свободных мест на индикаторе

### 2.3. Структура Student

```c
typedef struct {
    BathroomStatus gender;                // Пол студента (MALE/FEMALE)
    double shower_duration;               // Время принятия душа (сек)
    double wait_start_time;               // Время начала ожидания
    double wait_end_time;                 // Время окончания ожидания
    bool entered;                         // Флаг успешного входа
    int student_id;                       // Уникальный идентификатор
} Student;
```

**Назначение:** Представляет студента и собирает статистику по его использованию ванной.

## 3. Основные функции

### 3.1. `openBathroom()` — Инициализация ванной комнаты

```c
void openBathroom(Bathroom* b, unsigned int capacity, unsigned int max_entries)
```

**Назначение:** Инициализирует структуру ванной комнаты, создает мьютекс и условную переменную.

**Алгоритм:**
1. Инициализация мьютекса `pthread_mutex_init()`
2. Инициализация условной переменной `pthread_cond_init()`
3. Установка начальных значений всех полей структуры

### 3.2. `printDoorStatus()` — Отображение индикатора на двери

```c
void printDoorStatus(Bathroom* b)
```

**Назначение:** Выводит текущее состояние индикатора на двери и количество свободных мест.

**Вывод:**
- Состояние индикатора: "НИКОГО НЕТ" / "В ВАННОЙ МУЖЧИНЫ" / "В ВАННОЙ ЖЕНЩИНЫ"
- Количество свободных мест: `total_cabins - occupied_cabins`

### 3.3. `isEntryAllowed()` — Проверка возможности входа

```c
static bool isEntryAllowed(Bathroom* b, Student* s)
```

**Логика работы:**

1. **Полная занятость:** Если все кабинки заняты (`occupied_cabins >= total_cabins`) — вход запрещён.

2. **Ванная пуста (EMPTY):** 
   - Если установлен `forced_switch_pending` и последний пол совпадает с полом студента — вход запрещён (требуется смена пола)
   - Иначе — вход разрешён

3. **Ванная занята:** Вход разрешён только студентам того же пола, что и `current_status`, и только если не установлен `forced_switch_pending` (иначе вход запрещён до полного освобождения ванной).

### 3.4. `enterBathroom()` — Вход в ванную

```c
bool enterBathroom(Bathroom* b, Student* s)
```

**Алгоритм:**

1. Захват `access_lock`, проверка флага `bathroom_open`
2. Фиксация времени начала ожидания (`wait_start_time`)
3. Захват `wait_lock` и цикл ожидания: пока не выполняется `isEntryAllowed()` и ванная открыта — `pthread_cond_wait(&availability_signal, &wait_lock)` с попеременным освобождением/захватом `access_lock`
4. После пробуждения: если ванная закрыта — возврат false
5. Фиксация времени окончания ожидания, установка `entered = true`
6. Если ванная была пуста — сброс/обновление `consecutive_entries`, сброс `forced_switch_pending` при смене пола, установка `current_status`
7. Инкремент `consecutive_entries`; при достижении `max_consecutive_entries` установка `forced_switch_pending = true`
8. Инкремент `occupied_cabins`, вывод о входе, вызов `printDoorStatus()`
9. Освобождение `access_lock`, затем `pthread_cond_broadcast()` под `wait_lock` и освобождение `wait_lock`

### 3.5. `exitBathroom()` — Выход из ванной

```c
void exitBathroom(Bathroom* b, Student* s)
```

**Алгоритм:**

1. Захват `access_lock`
2. Декремент `occupied_cabins`
3. Если ванная освободилась (`occupied_cabins == 0`): сохранение пола в `last_gender`, сброс `current_status` в `EMPTY`
4. Вывод информации о выходе и вызов `printDoorStatus()`
5. Освобождение `access_lock`, затем захват `wait_lock`, `pthread_cond_broadcast(&availability_signal)`, освобождение `wait_lock`

### 3.6. `studentRoutine()` — Функция потока студента

```c
void* studentRoutine(void* arg)
```

**Назначение:** Функция, выполняемая каждым потоком студента.

**Алгоритм:**
1. Получение данных студента из аргумента
2. Попытка войти в ванную (`enterBathroom()`)
3. Если вход успешен:
   - Принятие душа (`usleep()` на время `shower_duration`)
   - Выход из ванной (`exitBathroom()`)
4. Завершение потока

### 3.7. `bathroomDay()` — Основная функция симуляции

```c
void bathroomDay(int student_count, double male_ratio, unsigned int capacity,
                 unsigned int max_entries)
```

**Алгоритм:**

1. **Инициализация:** Вывод параметров симуляции, вызов `openBathroom()`, вывод начального состояния двери
2. **Создание потоков:** Для каждого студента создаётся отдельный поток через `pthread_create()` с функцией `studentRoutine` и указателем на структуру `Student`
3. **Ожидание завершения:** Ожидание завершения всех потоков через `pthread_join()` — симуляция продолжается, пока все студенты не попытаются войти и выйти
4. **Завершение:** Установка `bathroom_open = false`, пробуждение всех ожидающих через `pthread_cond_broadcast()` (для корректного выхода потоков, оставшихся в ожидании)
5. **Статистика:** Подсчёт вошедших по полам и среднего времени ожидания по полю `entered` и разнице `wait_end_time - wait_start_time`
6. **Очистка:** Освобождение массивов потоков и студентов, уничтожение мьютексов и условной переменной

## 4. Механизм предотвращения голодания

В реализации применён механизм справедливости на основе счётчика последовательных входов:

**Механизм:**
- После `max_consecutive_entries` входов одного пола подряд устанавливается флаг `forced_switch_pending`
- При установленном флаге студенты того же пола не могут войти ни в пустую ванную (проверка в `isEntryAllowed`: при EMPTY и `last_gender == gender` вход запрещён), ни в занятую тем же полом (вход запрещён, пока флаг установлен)
- Флаг сбрасывается при входе студента противоположного пола в пустую ванную (в `enterBathroom`: при смене пола относительно `last_gender` выполняется `forced_switch_pending = false`)
- Это обеспечивает чередование полов и снижает риск голодания одного из полов

**Пример:** Мужчины входят 5 раз подряд → `forced_switch_pending = true`. Ванная освобождается → мужчины не могут войти в пустую ванную. Входит женщина → флаг сбрасывается при следующей смене пола.

## 5. Синхронизация потоков

### 5.1. Мьютексы (`pthread_mutex_t`)

**Использование:**
- `access_lock` — защищает доступ к полям структуры `Bathroom` (занятость, статус, счётчики). Захватывается при проверке `isEntryAllowed()`, изменении состояния при входе/выходе и при вызове `printDoorStatus()`.
- `wait_lock` — используется совместно с условной переменной: захватывается при вызове `pthread_cond_wait()` и `pthread_cond_broadcast()`.

**Функции:**
- `pthread_mutex_init()` — инициализация
- `pthread_mutex_lock()` / `pthread_mutex_unlock()` — захват и освобождение
- `pthread_mutex_destroy()` — уничтожение

### 5.2. Условная переменная (`pthread_cond_t`)

**Использование:**
- `availability_signal` — ожидание изменения состояния ванной; вызывается `pthread_cond_wait(&availability_signal, &wait_lock)` (освобождается именно `wait_lock`).
- После выхода из ванной или при обновлении состояния вызывается `pthread_cond_broadcast()` под `wait_lock`, чтобы разбудить ожидающие потоки.

**Функции:**
- `pthread_cond_init()` — инициализация
- `pthread_cond_wait()` — ожидание (передаётся `wait_lock`)
- `pthread_cond_broadcast()` — пробуждение всех ожидающих
- `pthread_cond_destroy()` — уничтожение

### 5.3. Работа с потоками

**Создание потока:**
```c
pthread_create(&threads[i], NULL, studentRoutine, &students[i]);
```

**Ожидание завершения:**
```c
pthread_join(threads[i], NULL);
```

- **Среднее время ожидания:** `(Σ(wait_end_time - wait_start_time)) / количество_вошедших` (время в секундах по `clock()`)

- **Мьютексы (`pthread_mutex_t`)** — на практике для защиты критических секций при доступе к общему ресурсу (ванной комнате)

- Отображение состояния индикатора на двери ванной комнаты
- Сбор статистики использования ресурса

Программа демонстрирует корректную работу многопоточного приложения с синхронизацией доступа к общему ресурсу и обеспечивает справедливое распределение доступа между различными группами пользователей.
