🔧 Системные вызовы и функции для работы с процессами

В версии программы, использующей процессы вместо потоков, необходимо:

  • создавать отдельные процессы для каждого студента (fork);
  • организовать разделяемую память для общих структур (mmap);
  • синхронизировать доступ с помощью мьютексов и условных переменных, способных работать между процессами (pthread_mutexattr_setpshared);
  • ожидать завершения всех процессов (waitpid).
  • (опционально) заменить образ процесса новой программой (exec), если требуется.

1. Создание и управление процессами

fork()

Прототипpid_t fork(void);
ОписаниеСоздаёт новый процесс (дочерний), который является почти полной копией родительского. Дочерний процесс получает собственное адресное пространство, но разделяемая память (отображённая с MAP_SHARED) остаётся общей.
Возвращаемое значение
  • в родителе: PID дочернего процесса ( > 0 )
  • в потомке: 0
  • при ошибке: -1 (errno установлен)

exit()

Прототипvoid exit(int status);
ОписаниеЗавершает вызывающий процесс. Закрываются все файловые дескрипторы, освобождается память (кроме разделяемой, которая остаётся доступной другим процессам). Код возврата status может быть получен родителем через waitpid.

waitpid()

Прототипpid_t waitpid(pid_t pid, int *status, int options);
Параметры
  • pid — идентификатор ожидаемого процесса. Если pid = -1, ожидается любой дочерний процесс.
  • status — указатель на переменную, куда будет записан код завершения (можно NULL).
  • options — флаги (например WNOHANG для неблокирующего режима). В задаче обычно 0 (блокировка).
ОписаниеПриостанавливает выполнение родительского процесса до изменения состояния указанного дочернего процесса (завершение, остановка). Возвращает PID завершившегося процесса или -1 при ошибке.

2. Разделяемая память между процессами

Для совместного использования структур Bathroom и массива students необходимо разместить их в разделяемой памяти, созданной с помощью mmap.

mmap()

Прототипvoid *mmap(void *addr, size_t length, int prot, int flags, int fd, off_t offset);
Параметры
  • addr — желаемый адрес начала отображения. NULL означает, что ядро выберет адрес самостоятельно.
  • length — размер отображаемой области в байтах (должен быть кратен размеру страницы, но ядро округляет).
  • prot — права доступа: PROT_READ (чтение), PROT_WRITE (запись), можно комбинировать через |.
  • flags — флаги отображения. Ключевые для задачи:
    • MAP_SHARED — изменения видны всем процессам, отобразившим ту же область.
    • MAP_ANON (или MAP_ANONYMOUS) — анонимная память, не связанная с файлом.
  • fd — файловый дескриптор. Для анонимной памяти указывается -1.
  • offset — смещение в файле (для анонимной памяти должно быть 0).
Возвращаемое значениеУказатель на начало отображённой области или MAP_FAILED (который равен (void*)-1) при ошибке.

Пример создания разделяемой памяти под массив студентов:

students = mmap(NULL, sizeof(St) * MAX_STUDENTS,
                PROT_READ | PROT_WRITE,
                MAP_SHARED | MAP_ANON,
                -1, 0);

Аналогично создаётся область для структуры ванной Br *b.

munmap()

Прототипint munmap(void *addr, size_t length);
ОписаниеУдаляет отображение памяти для указанного диапазона адресов. После вызова обращаться к этой памяти нельзя. Возвращает 0 при успехе, -1 при ошибке.
ПрименениеВ родительском процессе после завершения всех дочерних процессов:
munmap(b, sizeof(Br));
munmap(students, sizeof(St) * MAX_STUDENTS);

3. Синхронизация между процессами

Мьютексы и условные переменные POSIX (pthread_mutex_t, pthread_cond_t) могут использоваться между процессами только при выполнении двух условий:

  1. Они должны располагаться в разделяемой памяти (созданной через mmap с MAP_SHARED).
  2. При инициализации необходимо указать атрибут PTHREAD_PROCESS_SHARED.

Настройка атрибутов мьютекса и условной переменной

ФункцияНазначение
pthread_mutexattr_init(&attr);Инициализирует объект атрибутов мьютекса значениями по умолчанию.
pthread_mutexattr_setpshared(&attr, PTHREAD_PROCESS_SHARED);Устанавливает атрибут «разделяемости между процессами».
pthread_condattr_init(&cattr);Инициализирует объект атрибутов условной переменной.
pthread_condattr_setpshared(&cattr, PTHREAD_PROCESS_SHARED);Аналогично для условной переменной.

Особенности реализации в программе

В представленном коде используются два мьютекса:

  • dataMutex — защищает данные состояния ванной (количество занятых кабинок, текущий пол, счётчики ожидания и т.д.).
  • condMutex — используется совместно с условной переменной cond для ожидания и сигнализации.

Функция canEnter() захватывает dataMutex, проверяет возможность входа и отпускает его перед возвратом. Функции enterBathroom и leaveBathroom работают с condMutex и внутри себя могут дополнительно захватывать dataMutex (например, leaveBathroom захватывает dataMutex для обновления состояния).

Цикл ожидания в enterBathroom выглядит так:

pthread_mutex_lock(&b->condMutex);
while (!canEnter(b, s)) {
    pthread_cond_wait(&b->cond, &b->condMutex);
}

При этом canEnter сама управляет dataMutex, поэтому после пробуждения проверка выполняется корректно. Такой подход возможен, но требует строгого соблюдения порядка блокировок во избежание взаимных блокировок. В leaveBathroom сначала захватывается condMutex, затем dataMutex — это гарантирует согласованность.

4. Инициализация в функции init()

В коде вызывается init(), которая выполняет:

  • создание разделяемой памяти для Br *b и St *students через mmap;
  • инициализацию двух мьютексов и условной переменной с атрибутом PTHREAD_PROCESS_SHARED;
  • установку начальных значений полей ванной.
void init() {
    // Ванная в разделяемой памяти
    b = mmap(NULL, sizeof(Br), PROT_READ|PROT_WRITE,
             MAP_SHARED|MAP_ANON, -1, 0);

    // Студенты в разделяемой памяти
    students = mmap(NULL, sizeof(St) * MAX_STUDENTS,
                    PROT_READ|PROT_WRITE,
                    MAP_SHARED|MAP_ANON, -1, 0);

    pthread_mutexattr_t attr;
    pthread_mutexattr_init(&attr);
    pthread_mutexattr_setpshared(&attr, PTHREAD_PROCESS_SHARED);

    pthread_condattr_t cattr;
    pthread_condattr_init(&cattr);
    pthread_condattr_setpshared(&cattr, PTHREAD_PROCESS_SHARED);

    pthread_mutex_init(&b->dataMutex, &attr);
    pthread_mutex_init(&b->condMutex, &attr);
    pthread_cond_init(&b->cond, &cattr);

    // Начальные значения
    b->cabins_total = BATHROOM_CAPACITY;
    b->cabins_used = 0;
    b->state = nobody;
    b->last_state = nobody;
    b->force_change = false;
    b->waiting_men = 0;
    b->waiting_women = 0;
    b->streak = 0;
    b->max_streak = MAX_STREAK_FOR_STATE;

    pthread_mutexattr_destroy(&attr);
    pthread_condattr_destroy(&cattr);
}

5. Вспомогательные функции

getpid()

Прототипpid_t getpid(void);
ОписаниеВозвращает идентификатор текущего процесса. Используется в программе для вывода PID студента в момент входа в ванную, что помогает отслеживать активность разных процессов.

nanosleep()

Прототипint nanosleep(const struct timespec *req, struct timespec *rem);
ОписаниеПриостанавливает выполнение текущего потока/процесса на заданное время. Используется для имитации времени пребывания студента в душе.

6. Запуск новых программ: семейство функций exec

Функции exec заменяют текущий образ процесса новой программой. В нашей задаче они не используются, потому что дочерние процессы продолжают выполнять ту же программу (функцию studentProcess). Однако в общем случае после fork часто вызывают exec для запуска другой программы.

Общие свойства exec

  • При успешном вызове exec не возвращает управление (образ процесса полностью заменён).
  • При ошибке возвращает -1 и устанавливает errno.
  • Функции различаются способом передачи аргументов и поиска исполняемого файла.
ФункцияПрототипОписание
execl int execl(const char *path, const char *arg, ..., NULL); Принимает список аргументов (как строки, оканчивающиеся NULL). Путь к программе задаётся полным или относительным путём.
execlp int execlp(const char *file, const char *arg, ..., NULL); Аналогична execl, но поиск программы осуществляется по переменной окружения PATH (как в shell).
execle int execle(const char *path, const char *arg, ..., NULL, char *const envp[]); Позволяет передать окружение для новой программы (массив строк envp).
execv int execv(const char *path, char *const argv[]); Аргументы передаются в виде массива строк, оканчивающегося NULL.
execvp int execvp(const char *file, char *const argv[]); Аналогична execv, но с поиском по PATH.
execvpe int execvpe(const char *file, char *const argv[], char *const envp[]); Расширение GNU: execvp + возможность передать окружение envp.

Пример использования execlp для запуска программы ls:

execlp("ls", "ls", "-l", NULL);

Если бы в нашей задаче требовалось запустить отдельную программу для каждого студента, код мог бы выглядеть так:

if (fork() == 0) {
    execl("./student_proc", "student_proc", student_arg, NULL);
    perror("exec"); // выполнится только при ошибке
    exit(1);
}
Примечание: в текущей реализации студенты моделируются функцией studentProcess, которая выполняется в том же бинарном файле. Вызов exec не требуется.

7. Очистка ресурсов в родительском процессе

После завершения всех дочерних процессов родитель обязан:

  • уничтожить мьютексы и условные переменные (pthread_mutex_destroy, pthread_cond_destroy);
  • освободить разделяемую память (munmap).
pthread_mutex_destroy(&b->dataMutex);
pthread_mutex_destroy(&b->condMutex);
pthread_cond_destroy(&b->cond);
munmap(b, sizeof(Br));
munmap(students, sizeof(St) * MAX_STUDENTS);

Документ содержит подробное описание всех функций, необходимых для реализации многопроцессной версии программы «Ванная комната».
