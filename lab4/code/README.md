Лабораторная работа №4 — «Сетевое взаимодействие в ОС UNIX»
===========================================================

Проект состоит из двух программ:
- `server` — сервер, моделирующий ванную комнату;
- `client` — клиент(ы)-«студенты», которые подключаются к серверу и пытаются войти в ванную.

Исходники лежат в текущей директории:
- `server.c`
- `client.c`
- `Makefile`
- `run.sh`

---

Сборка
------

Находясь в каталоге `lab4/code`, выполните:

```bash
make
```

Будут собраны два исполняемых файла:
- `server`
- `client`

Для полной очистки:

```bash
make clean
```

---

Быстрый запуск (автоматически)
------------------------------

Самый простой способ запустить оба компонента — использовать готовый скрипт:

```bash
./run.sh
```

Что делает `run.sh`:
1. Освобождает порт 4567 (`fuser -k 4567/tcp`).
2. Стартует `./server` в фоне.
3. Даёт серверу подняться (`sleep 1`).
4. Запускает клиента `./client` в режиме нескольких студентов по умолчанию.
5. По завершении клиентов останавливает сервер.

Вывод сервера и клиента будет показан в том же терминале.

---

Ручной запуск сервера и клиентов
--------------------------------

### 1. Запуск сервера

По умолчанию сервер слушает IP `0.0.0.0` и порт из диапазона `4567–4587`.

Простейший запуск (после `make`):

```bash
./server
```

Сервер выведет строку вида:

```text
Сервер: 0.0.0.0:4567, кабинок 3, макс входов 5
```

Это означает:
- порт, на котором слушает сервер;
- количество кабинок в ванной (`3`);
- максимальное количество последовательных входов одного пола (`5`).

Также есть цели в `Makefile`:

```bash
make run-server
```

(при необходимости отредактируйте параметры по месту).

### 2. Запуск клиентов

Клиенты подключаются по TCP к тому же хосту и порту, который использует сервер.

#### 2.1. Многократный запуск (несколько студентов)

В `client.c` реализован режим `--multi`, позволяющий сразу запустить несколько студентов:

```bash
./client 127.0.0.1 --multi 20 50
```

Где:
- `127.0.0.1` — IP сервера (если сервер запущен локально);
- `20` — количество студентов;
- `50` — процент мужчин (остальные — женщины).

То же самое можно сделать через цель `Makefile`:

```bash
make run-clients
```

#### 2.2. Один конкретный студент

Можно запустить одного клиента с заданным ID и полом:

```bash
./client 127.0.0.1 1 M   # студент №1, мужчина
./client 127.0.0.1 2 F   # студент №2, женщина
```

Для удобства есть цели:

```bash
make run-male
make run-female
```

---

Типичный сценарий работы
------------------------

1. Собрать проект:

```bash
make
```

2. В одном терминале запустить сервер:

```bash
./server
```

3. В другом терминале запускать клиентов:

```bash
./client 127.0.0.1 --multi 20 50
```

или пользоваться скриптом:

```bash
make run
```

После выполнения вы увидите на стороне сервера лог входов/выходов клиентов и финальную статистику (сколько всего вошло, мужчин/женщин и т. д.).
