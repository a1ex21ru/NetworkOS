Лабораторная работа №4 — Защита

Сетевые сокеты на C (TCP, многоклиентский сервер с синхронизацией)

1. Общая архитектура

  • Протокол: TCP (SOCK_STREAM)
  • Семейство адресов: IPv4 (AF_INET)
  • Модель обработки клиентов на сервере: один поток на клиента (pthread)
  • Синхронизация доступа к разделяемым данным: мьютексы + условные переменные
  • Клиент: многопоточное приложение (каждый студент — отдельный поток)
  • Обмен данными: структуры фиксированного размера (msg_t)

2. Ключевые моменты настройки сервера

setsockopt() + SO_REUSEADDR Сервер 

int opt = 1;
setsockopt(server_fd, SOL_SOCKET, SO_REUSEADDR, &opt, sizeof(opt));

Назначение: позволяет повторно использовать локальный порт после закрытия сокета, даже если он находится в состоянии TIME_WAIT.

Почему важно: без этой опции при перезапуске сервера в течение 2-4 минут после остановки возникнет ошибка Address already in use.

Детали:

  • Вызывается до bind().
  • Не отменяет TIME_WAIT, а разрешает новому сокету занять тот же порт.
  • Не защищает от конфликтов двух одновременно работающих серверов.

INADDR_ANY Сервер 

address.sin_addr.s_addr = INADDR_ANY;

Означает, что сервер принимает соединения на всех доступных сетевых интерфейсах. Эквивалент 0.0.0.0.

Когда используют: если нужно, чтобы сервер был доступен из любой сети (включая localhost, внутреннюю сеть, внешний IP).

Альтернатива: указать конкретный IP (например, 127.0.0.1) — тогда сервер будет слушать только этот интерфейс.

htons() — порядок байтов Общее 

address.sin_port = htons(PORT);

Host TO Network Short — преобразует 16-битное значение из порядка байтов хоста (little‑endian на x86) в сетевой порядок (big‑endian), обязательный для всех протоколов TCP/IP.

Аналоги: htonl() (32 бита), ntohs(), ntohl() для обратного преобразования.

Что будет без htons? Порт будет интерпретирован неверно на другой архитектуре, соединение не установится.

3. Функции для работы с сокетами (TCP)

socket() Общее 

int socket(int domain, int type, int protocol);
ПараметрЗначение
domainAF_INET — IPv4
typeSOCK_STREAM — потоковый сокет (TCP)
protocol0 — выбор протокола по умолчанию (TCP)

Возвращает: файловый дескриптор нового сокета или -1 при ошибке.

Примечание: после успешного создания сокет не привязан к адресу.

Сервер Используется для создания слушающего сокета.

Клиент Используется для создания сокета для подключения к серверу.

bind() Сервер 

int bind(int sockfd, const struct sockaddr *addr, socklen_t addrlen);
ПараметрОписание
sockfdдескриптор сокета (от socket)
addrуказатель на структуру с адресом (например, struct sockaddr_in)
addrlenразмер структуры адреса

Назначение: привязывает сокет к локальному IP-адресу и порту.

Возвращает: 0 при успехе, -1 при ошибке.

Особенности: если порт уже используется другим процессом (и нет SO_REUSEADDR), bind завершится ошибкой.

listen() Сервер 

int listen(int sockfd, int backlog);
ПараметрОписание
sockfdдескриптор сокета (привязанного к адресу)
backlogмаксимальная длина очереди входящих соединений

Назначение: переводит сокет в пассивный режим ожидания подключений.

Возвращает: 0 при успехе, -1 при ошибке.

Важно: backlog — это размер очереди ещё не принятых соединений, а не максимальное число клиентов.

accept() Сервер 

int accept(int sockfd, struct sockaddr *addr, socklen_t *addrlen);
ПараметрОписание
sockfdдескриптор слушающего сокета (после listen)
addrуказатель на структуру для заполнения адреса клиента (может быть NULL)
addrlenуказатель на размер структуры адреса (должен быть инициализирован)

Назначение: извлекает первое соединение из очереди и создаёт новый сокет для общения с клиентом.

Возвращает: дескриптор нового сокета (для данного клиента) или -1 при ошибке.

Блокировка: accept блокируется, пока нет ожидающих соединений.

connect() Клиент 

int connect(int sockfd, const struct sockaddr *addr, socklen_t addrlen);

Используется в клиенте.

ПараметрОписание
sockfdдескриптор сокета клиента
addrадрес сервера (IP и порт)
addrlenразмер структуры адреса

Назначение: устанавливает соединение с сервером.

Возвращает: 0 при успехе, -1 при ошибке.

send() Общее 

ssize_t send(int sockfd, const void *buf, size_t len, int flags);
ПараметрОписание
sockfdдескриптор сокета
bufуказатель на буфер с данными
lenколичество байт для отправки
flagsфлаги (обычно 0)

Назначение: передаёт данные через сокет.

Возвращает: количество отправленных байт (может быть меньше len) или -1 при ошибке.

Примечание: для гарантированной отправки всего буфера нужно вызывать в цикле.

Сервер отправляет обновления состояния.

Клиент отправляет запросы (JOIN, LEAVE, STATS).

recv() Общее 

ssize_t recv(int sockfd, void *buf, size_t len, int flags);
ПараметрОписание
sockfdдескриптор сокета
bufуказатель на буфер для приёма данных
lenразмер буфера
flagsфлаги (обычно 0)

Назначение: принимает данные из сокета.

Возвращает:

  • >0 — количество принятых байт
  • 0 — удалённая сторона закрыла соединение
  • -1 — ошибка

Сервер принимает запросы от клиентов.

Клиент принимает обновления состояния от сервера.

close() Общее 

int close(int fd);

Назначение: закрывает файловый дескриптор сокета. При этом TCP‑соединение завершается (переходит в состояние FIN‑WAIT/TIME‑WAIT).

Возвращает: 0 при успехе, -1 при ошибке.

Сервер закрывает клиентский сокет после обработки.

Клиент закрывает сокет после завершения сеанса.

4. Вспомогательные функции для адресации

inet_ntop() Сервер 

const char *inet_ntop(int af, const void *src, char *dst, socklen_t size);

Назначение: преобразует бинарный IP-адрес (например, из sin_addr) в строку.

ПараметрОписание
afAF_INET для IPv4
srcуказатель на структуру с адресом (например, &addr.sin_addr)
dstбуфер для строки (например, char ip[INET_ADDRSTRLEN])
sizeразмер буфера

Возвращает: указатель на dst или NULL при ошибке.

getpeername() Сервер 

int getpeername(int sockfd, struct sockaddr *addr, socklen_t *addrlen);

Назначение: получает адрес удалённой стороны, подключённой к сокету.

Возвращает: 0 при успехе, -1 при ошибке.

5. Функции для работы с потоками (pthread)

pthread_create() Общее 

int pthread_create(pthread_t *thread, const pthread_attr_t *attr,
                   void *(*start_routine)(void *), void *arg);

Назначение: создаёт новый поток, который выполняет функцию start_routine с аргументом arg.

Возвращает: 0 при успехе, код ошибки при неудаче.

Сервер создаёт поток для каждого подключившегося клиента.

Клиент создаёт потоки-студенты.

pthread_detach() Сервер 

int pthread_detach(pthread_t thread);

Назначение: переводит поток в отсоединённое состояние — после завершения его ресурсы будут освобождены автоматически.

Использование: чтобы не хранить идентификаторы потоков и не вызывать pthread_join.

pthread_join() Клиент 

int pthread_join(pthread_t thread, void **retval);

Назначение: ожидает завершения указанного потока.

Использование в клиенте: главный поток ожидает завершения всех студентов.

pthread_mutex_init / lock / unlock / destroy Общее 

int pthread_mutex_init(pthread_mutex_t *mutex, const pthread_mutexattr_t *attr);
int pthread_mutex_lock(pthread_mutex_t *mutex);
int pthread_mutex_unlock(pthread_mutex_t *mutex);
int pthread_mutex_destroy(pthread_mutex_t *mutex);

Назначение: управление мьютексами для защиты критических секций.

Особенность: все функции возвращают 0 при успехе или код ошибки.

Сервер защищает общее состояние ванной.

Клиент используется для синхронизации вывода в консоль.

pthread_cond_init / wait / broadcast / destroy Сервер 

int pthread_cond_init(pthread_cond_t *cond, const pthread_condattr_t *attr);
int pthread_cond_wait(pthread_cond_t *cond, pthread_mutex_t *mutex);
int pthread_cond_broadcast(pthread_cond_t *cond);
int pthread_cond_destroy(pthread_cond_t *cond);

Назначение: условные переменные для организации ожидания выполнения некоторого условия.

pthread_cond_wait: атомарно разблокирует мьютекс и переводит поток в сон до получения сигнала. После пробуждения мьютекс снова захватывается. Всегда вызывается в цикле while(!condition) из-за возможных ложных пробуждений.

pthread_cond_broadcast: будит все потоки, ожидающие на данной условной переменной.

6. Вспомогательные функции (время, случайные числа, файлы)

time() Общее 

time_t time(time_t *tloc);

Назначение: возвращает текущее календарное время в секундах с 1970-01-01.

Используется для замера времени ожидания (wait_start).

Сервер может использовать для статистики.

Клиент фиксирует время отправки запроса.

sleep() / usleep() Клиент 

unsigned int sleep(unsigned int seconds);
int usleep(useconds_t usec);

sleep — приостанавливает выполнение потока на заданное число секунд (имитация пребывания в ванной).
usleep — на микросекунды (используется для создания интервала между запусками потоков).

rand() / srand() Клиент 

int rand(void);
void srand(unsigned int seed);

Назначение: генерация псевдослучайных чисел для определения пола и времени в душе. srand(time(NULL)) инициализирует генератор.

fopen() / fscanf() / fclose() Клиент 

FILE *fopen(const char *path, const char *mode);
int fscanf(FILE *stream, const char *format, ...);
int fclose(FILE *stream);

Назначение: чтение конфигурации (IP-адреса сервера) из файла config.txt.

7. Протокол взаимодействия (структура msg_t)

typedef struct {
    msg_type_t type;        // MSG_JOIN, MSG_LEAVE, MSG_UPDATE, MSG_STATS
    int id;                 // идентификатор студента
    int gender;             // 0=MALE, 1=FEMALE
    long wait_start;        // время отправки запроса на вход
    long bath_time;         // время в ванной (при выходе)
    int occupied;           // занято мест
    int capacity;           // всего мест
    int streak_gender;      // пол текущей серии (-1 если не определён)
    int streak_used;        // сколько вошло подряд
    int remaining_male;     // осталось мужчин (не зашедших)
    int remaining_female;   // осталось женщин
    long total_wait_time;   // суммарное время ожидания
    long total_bath_time;   // суммарное время в ванной
    int entered_count;      // сколько всего вошло
    int male_entered;       // из них мужчин
    int female_entered;     // женщин
} msg_t;

Сообщения фиксированного размера передаются через send/recv.

8. Возможные вопросы преподавателя

❓ Почему нужен setsockopt с SO_REUSEADDR?
Чтобы избежать ошибки "Address already in use" после перезапуска сервера, когда порт находится в состоянии TIME_WAIT.
❓ Что такое INADDR_ANY?
Это 0.0.0.0 — сервер слушает на всех сетевых интерфейсах.
❓ Что будет, если не вызвать htons?
Порт будет интерпретирован в порядке байтов хоста (little-endian), что не соответствует сетевому стандарту (big-endian). Соединение может не установиться.
❓ Что возвращает recv при отключении клиента?
0.
❓ Почему используется TCP, а не UDP?
Требуется надёжная доставка и сохранение порядка сообщений; состояние соединения важно.
❓ Почему в pthread_cond_wait используют цикл while, а не if?
Из-за ложных пробуждений (spurious wakeups) и возможности изменения условия несколькими потоками. Проверка в цикле гарантирует, что условие действительно выполнено.
❓ Может ли возникнуть взаимная блокировка (deadlock)?
В данной реализации используется один мьютекс, защищающий все общие данные. Нет вложенных блокировок, поэтому deadlock исключён.
❓ Что произойдёт при SIGPIPE во время send?
Если клиент разорвал соединение, send может вызвать сигнал SIGPIPE, который по умолчанию завершает процесс. Нужно либо обработать сигнал, либо использовать флаг MSG_NOSIGNAL.

9. Подводные камни реализации

  • Отсутствие SO_REUSEADDR — ошибка при перезапуске сервера.
  • Блокирующий accept — сервер не может завершиться, пока нет подключений.
  • recv может вернуть меньше данных, чем ожидалось — требуется циклический приём.
  • Необработанный SIGPIPE — аварийное завершение при send в закрытый сокет.
  • Игнорирование возвращаемых значений — пропуск ошибок.
  • Потеря сигнала условной переменной — если не проверить условие до wait.
  • Утечка памяти — нужно освобождать аргумент, переданный в поток (int *fd_ptr).

10. Возможные улучшения

  • Использование select / poll / epoll для одного потока на много клиентов.
  • Пул потоков вместо создания потока на каждого клиента.
  • Graceful shutdown (обработка SIGINT, завершение всех потоков).
  • Таймауты на чтение/запись (через setsockopt или alarm).
  • Верификация целостности сообщений (например, добавить длину).
